Design Patterns Strategy ->  é criada uma interface para metodos que se tornam recorrentes, passando o metodo apenas para ser sobreescrito dependendo da situação assim não sendo preciso a criação desse metodo a cada vez que precisa dele.

Chain of responsibility ->  é um padrão de projeto que permite passar a solicitação ao longo da cadeia de handlers em potencial até que um deles lide com a solicitação.

Template Method -> Faz parte da herança, se existe uma classe que tem atributos que podem ser reutilizados podemos usar essa classe como template para outras.

State -> é o estado que a classe se encontra de acordo com seu comportamento, quando é executado alguma função ele muda de estado.

Pattern command -> é o comando sendo executado por uma classe através de um metodo ao invés do main.

Listener é basicamente as classes sendo chamadas sem precisar chama-las uma por uma, criamos uma lista e passamos todas as classes para ser executadas.
